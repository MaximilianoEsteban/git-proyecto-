Areas de GIT

Working Directory

El area de trabajo corresponde a la carpeta del proyecto, donde vamos a realizar la modificaciones que incluyen la creacion, eliminacion y edicion de archivos. Normalmente vamos a trabajar con un editor de codigo y muy rara vez utilizaremos la linea de comandos.

  • dir || ls Muestra el contenido del directorio actual.
  • mkdir <name> Crea una carpeta con el nombre especificado.
  • cd <path> Nos redirige al directorio seleccionado.
  • git init Inicializa un repositorio de git.
  • rm -rf .git Elimina el repositorio de git.

Staging Area (INDEX)

El area de control de cambios es la encargada de realizar las capturas del codigo (SNAPSHOT) para que luego sean almacenadas en el repositorio cuando hagamos una confirmacion. Es importante tener en cuenta que solamente podremos reestablecer los archivos a los que le hayamos hecho seguimiento.

  • git status muestra el estado de los archivos de Working Directory en relacion a los registrado en el INDEX.
    • untracked archivo al que no se le esta haciendo seguimiento
    • staged archivo agregado al INDEX pero que no posee un commit.
    • commited archivo cuyos cambios ya fueron registrados en un commit.
    • modified archivo cuyo contenido difiere con lo registrado en el INDEX
  • git add <file> agrega el contenido indicado al INDEX para hacerle seguimiento.
  • git blame <file> muestra los datos de autor, commit, ademas de la fecha y hora de confirmacion; asi como tambien los cambios pendientes de un archivo.
  • git rm --cached <file> quita del INDEX el contenido especificado (untracked)
  • git restore --staged <file> restaura el estado del INDEX para los archivos seleccionados

Repository

Es el area donde se almacenan todas las confirmaciones de cambios en archivos de formato BLOB (Binary Large Object File) identificados con un numero de HASH y a los cuales podemos acceder a traves del historial, permitiendonos ver todas las modificaciones realizadas dentro de nuestro proyecto.

  • git commit abre el editor de codigo para realizar la confirmacion de los cambios registrados en el INDEX.
  • git log muestra el historial de cambios ordenado cronologicamente de manera descendente.
  • git diff <ref> compara y muestra las diferencia entre el Working Directory y el commit especificado.
  • git checkout <ref> mueve el apuntador HEAD a la posicion del commit especificado.
  • git clone <url> crea una copia local de un proyecto almacenado en un repositorio remoto.

Apuntadores de GIT

HEAD

Apuntador movil que permite al usuario desplazarse dentro del historial de cambios. Se utiliza como referente de la posicion actual dentro del arbol de confirmaciones y se puede utilizar de la siguiente manera: HEAD^ o HEAD~2 para indicar a cuantos commit ancestros o anteriores debemos movernos.

BRANCH

Apuntador dinamico que nos permte regresar al ultimo commit de una historia determinada, se desplaza automaticamente cada vez que generamos un commit. Podemos crear las ramas que sean necesarias, pero a pesar de las confirmaciones, todas van a tener un commit en comun.
